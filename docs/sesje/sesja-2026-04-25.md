### Lekcje z dnia

1. **Sprzeczne walidacje w prompcie Codex = build-fixy.** Limit 20 w domain + "+N kolejnych" w UI to dwie sprzeczne specyfikacje. Następne prompty: jedno źródło prawdy per zasada, jawne wykluczenie sprzeczności.

2. **Codex zapomina o importach.** Dwa razy w 4C2/C (produkcja + test). Następne prompty: lista wymaganych importów wymieniona jawnie w prompcie.

3. **Codex pracuje offline w swoim środowisku.** `git fetch origin` zwraca błąd, brak push, lokalna gałąź ad-hoc. Konsekwencja: patch generowany od baseline (zazwyczaj main), nie od HEAD gałęzi której dotyczy. Aplikacja patcha do gałęzi która ewoluowała → konflikty od pierwszej linii.

4. **Patch z UI Codex ma blob hash w nagłówku.** Łatwa diagnoza: `git ls-tree HEAD <plik>` pokazuje obecny hash, porównanie z `index hash..hash` w patchu mówi czy patch jest "od czego" wyszedł.

5. **Droga zastępcza dla niedziałającego patcha:** QG czyta aktualny plik (przez Code zapisanego do txt - obejście limitów UI dla długich outputów), pisze prompt ze str_replace zawierającym dokładny stary blok ↔ dokładny nowy blok. Bez patcha, bez zgadywania.

6. **Codex w iteracji 2 skasował ~15 testów spoza zakresu prompta.** Jego patch testowy ignorowaliśmy - testy dodane ręcznie z prompta QG. Lekcja: każdy patch Codex wymaga przeglądu listy plików zmienionych vs. zakres prompta przed użyciem.

7. **Asercje typu `assertTrue(x.size() >= 0)` są bezużyteczne.** Codex wygenerował taką w `shouldEmitPermanentFailureAlertWithAffectedAccounts`. Zawsze prawda, test niczego nie sprawdza. QG poprawił na `assertTrue(contains("crypto"))` + `assertTrue(contains("forex"))`.

8. **Potwierdzenie merge wymaga raportu Code z `git log`, nie tylko słów Architekta** (zasada precedensu 24.04). Po "zrobiłem" Architekta - Code uruchomił `git fetch + checkout main + pull + log`, dopiero wtedy QG zaktualizował pamięć i ten dokument.
